### 3.2 参数输入改造（重点测试）

**操作路径**：

1. 登录 → 侧栏 "设计优化"
2. 上传一个有效的 `.sldprt` 文件（或者从历史任务进入一个已完成任务）
3. 等待 AI 提取参数（页面会出现 `FloatingConfigButton` 漂浮按钮）
4. 点击漂浮按钮，弹出 **设计优化参数配置 Modal**

**Modal 内逐项验证**：

| 检查项 | 期望表现 |
|--------|----------|
| `优化方法 (method)` 行 | 仍是 **下拉框** (GA / HA)（这个不改） |
| `最大应力 (permissible_stress)` 行 | 仍是 **数字输入框**（这个不改） |
| `代数 (generations)` 行 | ✅ **数字输入框** + 💡推荐 + `[应用]` 或 `✓已应用` 徽章 |
| `种群数量 (population_size)` 行 | ✅ **数字输入框** + 💡推荐 + `[应用]` 或 `✓已应用` 徽章 |

**互动验证**：

- **改变勾选数量**：在表格中取消勾选几个优化变量，观察 `代数` / `种群数量` 旁的推荐数字：
  - 5–8 个变量时推荐 `30` / `20`
  - 8 个以上时推荐 `50` / `30`
  - 少于 5 个时回到默认 `6` / `7`
- **手动输入**：在 `代数` 输入框敲一个 `25`，徽章应变成 `💡 推荐 30 [应用]`
- **一键应用**：点 `[应用]`，输入框立即填入 `30`，徽章变为 `✓ 已应用`
- **min/max 约束**：试着输入 `0` 或 `999`，浏览器应阻止或显示原生数字校验红框

**提交验证**：

- 调整完成后点 `提交并开始运行`，**打开 Network 面板**，查看请求 `/tasks/optimize/submit-params` 的 payload：

```json
{
  "conversation_id": "...",
  "task_id": ...,
  "params": {
    "大半径": {"min": ..., "max": ...},
    ...
    "generations":     {"value": 25},          // ← 注意是数字，不是字符串
    "population_size": {"value": 20},
    "permissible_stress": {"value": 355000000},
    "method": {"value": "GA"}
  }
}
```

> ⚠️ **关键观察点**：`generations` / `population_size` 的 `value` 应为**数字类型**（之前下拉版本提交的是字符串 `"30"`）。如果后端原本接受字符串，这里改成数字后**可能会兼容性失败** — 请重点核对。

### 3.3 前后对比面板 MVP（重点测试）

**操作路径**：

1. 在设计优化页面**完成一轮完整优化**（上传 → AI 提取参数 → 提交 → 等待 best 出现）
2. 屏幕**最下方** ConversationDisplay 与上传栏之间应出现一条 **绿色标题** 的 `优化前后对比` 折叠面板

**面板内容逐项验证**：

| 检查项 | 期望表现 |
|--------|----------|
| 折叠头 | 左侧绿色 ⬇ 图标 + "优化前后对比" + "已迭代 N 代 · M 次仿真"；右侧 ⬆/⬇ 切换 |
| 点击折叠头 | 内容区平滑收起/展开 |
| 双 ModelSlot 区 | 左 "优化前 (Before) [初始]"，右 "优化后 (After) [gen N]"；中间 ⇄；目前显示截图（若有 screenshot parts）或灰色 Box 占位 |
| 体积数值卡 | 左侧灰色划掉初始值（如 `848.00 cm³`） → 右侧绿色加粗最优值（如 `638.00 cm³`） + ⬇ 百分比 |
| 应力数值卡 | 同上；若 after > 许用应力，应显示 🟠 `超限` 警示 |
| 算法配置一览 | `算法 HA · 代数 30 · 种群 20 · 许用应力 355.0 MPa` |
| 底部 ℹ 信息条 | 提示"待后端补 `initial_stl_file` 与 `stl_file` 字段后启用真实 STL 渲染" |

**单位换算检查**（重要 — 这是我从你日志样例反推的，可能不准）：

- 后端日志 `体积=0.000848` → 我假设单位是 **m³**，前端显示 `848.00 cm³`（乘 1e6）
- 后端日志 `应力=3964707328.000000` → 假设是 **Pa**，前端显示 `3964.7 MPa`（除以 1e6）

> ⚠️ 如果单位不对（比如体积其实是 cm³，应力是别的单位），请贴一行后端的真实输出我来调整 `formatVolume` / `formatStress`。

## 4. 可能出现的问题与排查清单

| 现象 | 可能原因 | 排查 |
|------|----------|------|
| 拉取后页面没变化 | 浏览器缓存了旧 bundle | `Ctrl+F5` 强制刷新；或检查 nginx web root 是否真的换了 |
| Modal 打开后参数行还是下拉框 | 部署的还是旧 dist | 看 dist/assets 文件名 hash，应为 `index-3e60b632.js` 系列 |
| 前后对比面板**完全不显示** | 日志解析失败（正则不匹配） | F12 → 切到对应 task → 右键 React DevTools 或在 console 跑 `JSON.stringify(window.lastMsg?.content?.slice(0,500))`；把片段贴给我 |
| 面板出现但数值卡显示 `—` | `仿真完成` 行没匹配上 | 同上，贴 content 前后 300 字符 |
| 推荐徽章数字始终是 `6 / 7` | 勾选数 < 5（推荐逻辑默认值） | 多勾选几个变量再观察 |
| `[应用]` 按钮点不动 | 该参数行未被勾选（disabled） | 确认 checkbox 状态 |
| 提交参数后后端报 422 / 类型错误 | generations/population_size 改成了数字，后端可能仍要求字符串 | 把响应 body 贴给我，我决定是否在提交时 `String(...)` 兼容 |
| 老优化任务历史进入后面板空白 | 历史消息 `content` 字段为空或被压缩 | 这是预期，老数据可能没完整日志 |
