## 2. 前端轻量化与 AiMessage 负载优化

结合 `pretext` 库的“分离测量缓存与热路径计算，规避无意义的高频 Reflow”的核心思想，我们排查了当前的大会话渲染逻辑，并落地了如下优化：

**已经落地的优化（Refactor 阶段）：**
- **抽离热路径的类型推断与图片去重**：之前 `AiMessage` 中的 `inferMessageType` (处理大量 includes 检测) 和 `uniqueImages` （Set 对象生成及图片去重）完全在 React 组件的 `useMemo` 或者 render block 中定义，尤其是随着上下文变长、流式逐字 update 时开销会被放大。
- 我们已经将其分离成纯函数架构至 `src/utils/messageUtils.js` (`inferMessageType`, `extractUniqueImages`, `safeParseMetadata`)，使每次 React render 中的引用与对象创建趋向于最简。

**针对 `pretext` 轻量化适配的可行建议**：
1. **纯粹测算预测与分层渲染**：当后续长文本（Markdown）、富代码块导致严重抖动时，可在完成流式输入前使用轻量化 Text 元素兜底，在结束流后再真正走 Markdown parsing/React-Syntax-Highlighter (类似 `pretext` “先布局，后正式装载”的思路)。
2. **纯算术虚拟预估**：`AiMessage` 可在服务器明确返回行高/行数 meta 信息的场景下，跳过 DOM 测量机制预分配高度，减少 React 的 Layout Thrashing。

## 5. 测试覆盖与容错机制说明

为了保障系统的稳定性与交互鲁棒性，前端已建立核心渲染链路的自动化测试套件（Vitest + React Testing Library）。

### 5.1 核心测试用例 (5/5 覆盖)
目前所有测试用例均 100% 通过（跑通命令：`npm run test`）。主要验证了以下三个层面的稳定性：
1. **消息路由精确度**：系统能够准确分析后端返回的消息特征，正确地将“普通对话”、“优化过程分析”和“3D建模展示”分发到对应的UI组件中。
2. **大模型失效容错（优雅降级）**：针对 AI 接口可能发生的异常（如由于配额耗尽或网络中断导致仅输出文字、未下发 3D 代码文件），前端不再因缺少依赖文件而陷入无限 Loading 锁死，而是能在秒级识别异常状态，并主动向用户展示降级提示（例如提示文件生成失败），保证系统体验不中断。
3. **排版格式防崩溃保护**：针对当前含有深度思考过程的大模型（其内部通常含有大量非标准 `<think>` 标签及混合代码），前端能够在渲染前级完成格式隔离与清洗。确保大模型凌乱的内部思考标签不会破坏原生的 Markdown 解析树，从而完美保护了核心数据表格和页面布局的呈现逻辑。

### 5.2 机制工程意义
上述机制填补了此前因 AI 大模型输出格式不可控导致的前端白屏、排版错乱等隐患，使得本系统的问答展示层达到了可稳健演示的工业标准。
